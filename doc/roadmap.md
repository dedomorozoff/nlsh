# Roadmap: NL Shell на Go + локальная `llama.cpp`

## 1) MVP scope
- Естественный язык -> команда shell.
- Linux-first.
- Локальная GGUF-модель через CGO-интеграцию `llama.cpp` (без HTTP/daemon).
- Режимы:
  - `ask` (только объяснение),
  - `suggest` (предложение команды),
  - `execute` (выполнение только после подтверждения).

## 2) Архитектура
- `InputParser` -> `PromptBuilder` -> `LlamaCgoEngine` -> `ResponseParser` -> `SafetyPolicyGate` -> `CommandExecutor`.
- Один процесс, загруженная модель переиспользуется в рамках сессии.
- Выход модели ограничен JSON-контрактом.

## 3) Этапы

### Этап A — Bootstrapping + CGO
- Инициализация Go модуля и CLI.
- Подключение `llama.cpp` submodule.
- Минимальный engine: load / generate / close.

### Этап B — Надежный NL -> Command
- Системный промпт + примеры.
- Строгая валидация JSON.
- Retry с repair-инструкцией на кривой output.

### Этап C — REPL/UX
- Интерактивный цикл.
- Показ команды, причины, уровня риска.
- Подтверждение на medium/high risk.

### Этап D — Контекст и качество
- Контекст cwd и последних команд.
- Ограничение контекста токен-бюджетом.

### Этап E — Hardening
- Таймауты и cancel.
- Контроль памяти и `Close()` discipline.
- Интеграционные тесты и базовые метрики.

## 4) Безопасность по умолчанию
- Ничего не запускать без explicit confirm.
- Блок опасных паттернов (`rm -rf /`, `mkfs`, `curl|sh` и т.д.).
- `--dry-run` по умолчанию на ранней стадии.

## 5) KPI MVP
- >=80% корректных команд на сценарном наборе.
- 0 автозапусков high-risk.
- Latency ответа ~2-4s на типовых запросах.
- Fallback в уточняющий вопрос при низкой уверенности.
